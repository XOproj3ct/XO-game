Main

• Invokes WELCOME_RULES function.

• User should enter any key then START_GAME will be invoked.

• Invokes START_GAME function.

• invokes exit after some point will be discussed later on.

START_GAME (core function)

• uses XO_ANIMATION function to run X-O animation as a loading screen.

• the game starts by using a while loop:

  • - set to 1
  • - invokes USER_INPUT_AND_UPDATES function
  • - PRINT_GAME_RESULT returns 1 —> else it returns 0
  • - increases end_game_counter for every loop by 1

• END_OR_NEW_GAME is invoked after the while loop breaking.

PRINT_GAME_RESULT

• It will be invoked after every USER_INPUT_AND_UPDATES invocation as shown in START_GAME docs.

• It executes its blocks of code according to playerX / playerO values.

• Using if/else it invokes JUST_THE_TABLE and the game result message then:

  • - Returns 1 if:
    • >> There is an update in playerX / playerO
    • >> end_game_counter = 9 and there is no winner; it’s considered a DRAW
  • - Else it returns 0 and the while loop in START_GAME continues.

• Breaks START_GAME game loop if returned 1 —> else it returns 0

UPDATE

• It is invoked in USER_INPUT_AND_UPDATES as discussed before.

• It accepts X_USER/O_USER as a params then does the following logic:

  • - Checks input validation.
  • - Checks if the input is already taken.

• Using invalid and taken :

  • - if invalid is set to 1 it alerts the user to Enter a valid number
  • - if taken is set to 1 it alerts the user that the number is Already taken
  • - if both are 0 , it updates the inputted position in the table to X or O based on the player's turn.

NOTE if invalid/taken is 1 so the flag and end_game_counter must be set to its previous value to avoid undefined behaviours and this logic is implemented in a nested if condition.

CHECK_WINNER

• Checks rows, columns, and diagonals, and then updates playerX/playerO to 1 if there’s a winner

END_OR_NEW_GAME

• It is invoked when the while loop in START_GAME is breaked

• It uses NEW_END to receive user input

  • - asks the user to input Y/y for a new game
  • - asks the user to press any key to exit

• If NEW_END = Y/y:

  • - RESET_GAME is invoked
  • - returns START_GAME

• else it invokes END_SCREEN

END_SCREEN

• Prints the outro of the game, Animation to the last winner, Production team members and bye message.

RESET_GAME

• Invoked if the user wants a new game ( NEW_END = Y/y )

• it resets all the variables
